1 · Install consent — the loud wall gets its dialog

Appears on weaver install something.weave when a GUI is present. The quiet surface is shown but not asked about — visibility is the point; consent friction is reserved for the ladder.

Decisions embedded here, for your veto:
  • Quiet ≠ hidden. Origins/providers/storage are always listed — the audit IS the trust story — but have no checkbox. Asking consent for "shows the time" trains people to click through.
  • The middle button is the design. "Install without command access" makes degraded install the easy safe choice — the widget runs, the loud capability stays off, the widget sees it as permission-denied. No all-or-nothing dark pattern.
  • Author must justify. The quoted "why" strings come from the manifest (capabilities: [{ id: "run-commands", reason: "…" }]). No reason string → check error. Agents will write honest reasons; humans will read them.
  • First-run disclosure for commands: the first actual command a widget runs is surfaced as a notification with the literal command line. Loud at grant, loud at first use.
  • Lineage is a first-class line under the title — remix culture starts in the install dialog.

    Decisions embedded here:
  • Rows lead with identity and liveness ("dancing to: Spotify", "21:14 left") — the manager should feel like the desktop's pulse, not Task Manager.
  • Cost is ambient, not shouty: a 60s sparkline + current figures per row, one desktop total in the header. Sorting/red-flagging only when something is an outlier.
  • Toggles pause, never uninstall. Uninstall/permissions/source/remix live one right-click deep per row.
  • "+ Conjure…" is the primary action — it opens a prompt box that hands off to your agent. The manager sells the Loop, not a file picker. ("Install .weave" is the secondary path.)
  • ⚠ badges persist on any widget holding a loud grant — the wall stays visible after install day.
